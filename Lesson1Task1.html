How to Write Perfect Software.

Lesson 1: What is Good Software?

TASK 1: Explain the difference between Waterfall and Agile.

A. The Waterfall System for Software Projects.

The Steps in WaterFall System:

  • Start of the project.
    1. Analysis
    2. Design
    3. Implementation
    4. Testing
    5. BugFixing
    6. Build , documentation, deployment.
  • End of the project.(1 month, 6 months, 1 year, 3 years
  • Bugfixing and maintenance

B. The Agile System for Software Projects

MODEL 1: traditional
  • Start of the project N Sprints.
    • Start of Sprint 1.
      1. Analysis
      2. Design
      3. Implementation
      4. Testing
      5. BugFixing
      6. Build, documentation
    • End of Sprint 1
    • ...
    • Start of Sprint N.
      1. Analysis
      2. Design
      3. Implementation
      4. Testing
      5. BugFixing
      6. Build, documentation
    • End of Sprint N.
  • End of the Project.
MODEL 2: TDD -- Test Driven Development
Start of the project Start of the Sprint 1 1) Analysis 2) Design and implement Unit Tests 3) Build, documentation End of the Sprint 1 Start of sprint 2 1) Analysis 2) Design and implement Unit Tests 3) Build, documentation End of the Sprint 2 ... ... ... End of the Project
MODEL 3: TDD and DDD (Domain Driven Development)
  • Start of the project
    • Start of the Sprint 1
      • 1) Analysis 2) DD Design and implement Unit Tests 3) Build, documentation
    • End of the Sprint 1
    • ...
    • Start of the Sprint N.
      • 1) Analysis 2) DD Design and implement Unit Tests 3) Build, documentation
    • End of the Sprint N.
  • End of the Project

C. Waterfall Versus Agile.

Waterfall Agile
One big sprint (The time of the project) Many sprints.(mostly 2 weeks per sprint)
Changing requirements is a big problem Changing requirements is always possible.
One big meeting and agreement A lot of meetings (every sprint).
Every step is done once Every step is done again every sprint.
Adaptions can be a problem. Adaptions are part of the process.
Most communication with stakeholders at the start. communication with stakeholders all the time. (users and developers)
